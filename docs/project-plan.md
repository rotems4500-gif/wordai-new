# תוכנית עבודה אדריכלית לפרויקט: Word AI Assistant

מסמך זה מפרט את תוכנית העבודה המקיפה להפיכת התוסף (Add-in) ממימוש מקומי/אישי (MVP) למוצר Production יציב, מאובטח וקל לתחזוקה.

## 1. ארכיטקטורה ואבטחה (Backend & Security)
הגישה הנוכחית של שמירת מפתחות API ב-`localStorage` וביצוע קריאות ישירות מהלקוח (Client-side) למנועי ה-AI מסוכנת ואינה מתאימה למוצר הפונה לקהל רחב.
*   **הקמת שרת תווך (Proxy Backend):** העברת כל הקריאות ל-Gemini ו-Perplexity דרך שירות ענן (למשל Firebase Cloud Functions או שרת Node.js).
*   **ניהול הרשאות (Authentication):** חיבור המשתמשים דרך Firebase Auth כך שרק משתמשים מורשים יוכלו לצרוך מכסות API.
*   **ביטול סקריפט סנכרון מקומי:** הסרת `sync-storage-to-local.mjs` לטובת משיכת קבצים פנימית (Dynamic Fetching) ישירות מהאחסון בענן (Firebase Storage) לאפליקציית הלקוח.

## 2. אופטימיזציה של ממשק Word (Office.js)
תהליכים ארוכים כמו "סריקת מסמך וביבליוגרפיה" עלולים להקפיא את חווית המשתמש או להיכשל בשל מגבלות זיכרון של חלונית העזר (Taskpane).
*   **עיבוד במנות (Chunking):** קריאת המסמך בחלקים במקום בבת אחת כדי לשמור על ממשק משתמש רספונסיבי ולמנוע קריסות של ה-Taskpane.
*   **שימור עיצוב מקורי:** סקירה ושיפור של מתודות ההזרקה (Insert logic) כך שהתוכן המוזרק (Retrieved/Generated) ישמור על העיצוב (Rich Text/HTML) ולא ישבור את מסמך היעד.
*   **תהליכי רקע:** שימוש ב-Web Workers לניהול משימות פרסור ארוכות.

## 3. מבנה ממשק משתמש וחווית משתמש (UI/UX)
*   **ניהול מצב (State Management):** החלפת ניהול מצב לוקאלי מפוזר בפתרון ריכוזי (לדוגמה Zustand או React Context מתקדם) כדי לתמוך במעברים חלקים בין ה"עורך", ל"מחקר", ול"הגדרות".
*   **משוב למשתמש (Feedback & Error Handling):** הוספת חיווי טעינה (Skeletons/Spinners) מפורט עבור כל קריאת AI, וכן טיפול שגיאות ברור (Toasts) במקרה של ניתוק רשת או שגיאת API.

## 4. איכות קוד והפצה (Quality & CI/CD)
*   **בדיקות אוטומטיות:** כתיבת בדיקות יחידה (Unit Tests) לפונקציות הליבה (למשל הפילטור והפרסור של תוצאות הביבליוגרפיה).
*   **תהליך בנייה אוטומטי (CI/CD):** הגדרת GitHub Actions שיריץ לינטר (Linter), יבנה את פרויקט ה-Vite, ויפרוס (Deploy) הכל (שרת סטטי + פונקציות ענן) באופן רציף.
*   **הגדרות סביבה:** הפרדה מוחלטת בין סביבות הפיתוח (development, staging, production).
